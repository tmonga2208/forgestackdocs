# Introduction to ForgeStack

Welcome to **ForgeStack**, a Firebase-focused stack template built with **Next.js**. ForgeStack simplifies the development of serverless applications by providing pre-configured Firebase integrations, custom hooks, and a scalable architecture.

## Why Use ForgeStack?

- 🔥 **Firebase-First** – Built to leverage Firebase Realtime Database, Firestore, Auth, and Storage.
- ⚡ **Optimized for Next.js** – Full support for SSR, ISR, and API routes.
- 🎨 **Theming Support** – Includes built-in light/dark mode.
- 🛠 **Custom Hooks** – Easy-to-use Firebase utility hooks.
- 🚀 **Production-Ready** – Scalable and well-structured codebase.

## Features

### 🔐 Authentication
Easily integrate Firebase Authentication using the built-in `useAuth.ts` hook.

### 📂 Database & Storage
Access **Firestore**, **Realtime Database**, and **Firebase Storage** with helper hooks (`useDatabase.ts`, `useRealtimeDB.ts`, `useStorage.ts`).

### 🎨 Theming
Built-in **dark mode** support with `next-themes`.

### ⚡ CLI for Quick Setup
Use `create-forgestack` CLI to bootstrap your project in seconds.

## Who is ForgeStack for?
ForgeStack is ideal for:
- Developers building **Firebase-powered applications**
- Teams that want a **pre-configured** Next.js & Firebase setup
- Anyone looking for a **scalable, serverless** architecture
